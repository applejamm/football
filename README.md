# Football · 中国体彩竞彩足球盘口工具集

> 一套自用的"**盘口抓取 + 漂移监控 + 决策辅助分析**"工具，
> 围绕中国体育彩票"竞彩足球"玩法（胜平负 / 让球胜平负 / 比分 / 总进球数）。

---

## 一、它是干什么的（Business Objective）

| 目标 | 说明 |
|---|---|
| **数据资产化** | 把"截图看赔率"换成"API 抓 + JSON 落盘"，每次抓取都是一份独立的时间快照 |
| **市场动态可视化** | 自动 diff 上次 vs 本次，输出"哪些选项被压热 / 压冷、变了多少" |
| **决策辅助** | 用同一份结构化数据，输出"市场画像 + 投注候选 + 资金分配"的分析报告 |

## 二、解决了什么用户问题（User Problem）

| 痛点 | 之前 | 现在 |
|---|---|---|
| 录入慢 | 截图后手抄，约 10 分钟/期 | API 一次拉 4 天 × 4 场，< 2 秒 |
| 数据零散 | 截图只能看不能算 | JSON 落盘，可比、可统计、可时序 |
| 看不到市场动向 | 只看到"此刻的赔率"，看不到"怎么变的" | diff 自动输出"压热/压冷"清单 |
| 决策凭感觉 | 没有结构化分析 | 有分析报告模板（市场画像 + 三档候选） |

## 三、5 秒快速上手

```bash
cd /Users/CursorProject/football

# 抓全部可见的比赛日（一次拉 4 天 × 4 场）
python3 fetch_odds.py

# 只抓某一天
python3 fetch_odds.py --day 260614

# 追盘口漂移：赛前每 15 分钟跑一次，每次自动 diff 上一次
python3 fetch_odds.py --day 260614 --diff-min-pct 1
```

每次运行后，**新文件 + 漂移清单**自动落盘，无需任何额外操作。

## 四、命名规则（核心约定，请勿改）

### 4.1 盘口数据

```
odds_<比赛日code>_<抓取日YYYYMMDD>-<时间HHMMSS>.json
```

例：`odds_260614_20260614-194448.json` =
> **"260614 这天的比赛盘口，2026-06-14 19:44:48 抓的"**

- `260614` = 体彩官方比赛日编号（即截图上"周日 06-14"角标）
- `20260614-194448` = 本地时间抓取的瞬间

按字母排序 = 按时间排序 → `ls odds_260614_*` 直接看时间线。

### 4.2 漂移清单

```
diffs/diff_<比赛日code>_<YYYYMMDD>-<HHMMSS>.json
```

抓取时间与对应的 `odds_*.json` 一致，便于 1:1 对照。

### 4.3 API 原始响应（可选）

```
raw/api_match_calc_<YYYYMMDD>-<HHMMSS>.json
```

只在 `--save-raw` 时生成，留作"字段变更追溯"用。

```json
          "odds": { "胜": 1.70, "平": 4.80, "负": 3.00 } },
        { "type": "比分",       "status": "已开盘",
          "odds": {
            "胜": { "1:0": 16.5, "2:0": 8.0, "胜其它": 3.80, ... },
            "平": { "0:0": 60.0, "1:1": 32.0, "平其它": 900.0, ... },
            "负": { "0:1": 90.0, ... , "负其它": 800.0 }
          } },
        { "type": "总进球数",   "description": "双方进球数之和", "status": "已开盘",
          "odds": { "0": 60.0, "1": 15.5, "2": 7.45, ... , "7+": 4.65 } }
      ]
    }
  ],
  "notes": [...]
}
```

### 字段约定

| 字段 | 含义 |
|---|---|
| `handicap` (int) | 让球数；正数 = 主队被让球，负数 = 主队让球，0 = 不让球 |
| `handicap_label` | 页面上的标签，如 `[-3]` `[+1]` `[0]` `[未]`（未开盘） |
| `status` | `已开盘` / `未开盘`（体彩 API 字段为 `Selling` 等英文，已转译） |
| 比分 `胜其它/平其它/负其它` | 未列出的更冷比分集合，例如 6 球以上比分 |
| 总进球数 `"7+"` | 7 球及以上（含点球） |

## 七、典型工作流

### 7.1 赛前 3 小时持续追盘

```bash
# 每 15 分钟跑一次（可放进 cron / launchd）
python3 fetch_odds.py --day 260614 --diff-min-pct 1
```

每次运行：
1. 抓最新盘口
2. 写入新文件 `odds_260614_<新时间>.json`
3. 与"同一比赛日下时间最新的旧文件"做 diff
4. 控制台打印"≥1% 变化的项"清单
5. 漂移详情落盘到 `diffs/diff_260614_<新时间>.json`

### 7.2 赛后复盘

```bash
ls odds_260614_*.json   # 拿到时间线
ls diffs/diff_260614_*  # 拿到每次的漂移记录
```

把这些时间序列与最终比分对照，可统计"哪些方向的资金流真的预测到了结果"。

### 7.3 决策分析

> 参考已有的 `analysis_260614_match009_010.md` 模板：
>
> 1. **市场画像**：每场比赛 5 个维度（强弱、胜负倾向、节奏、最热比分、冷门信号）
> 2. **跨市场叙事一致性**：让球 ↔ 比分 ↔ 总进球数是否互相印证
> 3. **三档候选**：稳（高命中）/ 均（性价比）/ 狂（搏冷门）
> 4. **资金分配**：稳约 40% / 均约 35% / 狂约 20% / 留白 5%
> 5. **风险提示**：包括返还率、独立性假设、市场水分等

## 八、命令行参数速查

| 参数 | 默认 | 说明 |
|---|---|---|
| `--day <code>` | 全部 | 只抓指定 matchDayCode（如 260614） |
| `--output-dir <path>` | 脚本所在目录 | 输出目录 |
| `--save-raw` | 关 | 同时保留 API 原始响应 |
| `--no-diff` | 关 | 不计算漂移 |
| `--diff-min-pct <N>` | 0 | 过滤变化绝对值小于 N% 的项 |

## 九、数据源与限制

| 项 | 说明 |
|---|---|
| 数据源 | 体彩官方 API `webapi.sporttery.cn`（移动端"足球比分计算器"使用的同款接口） |
| 鉴权 | 无；带 `Referer: https://m.sporttery.cn/` 即可 |
| 响应时间 | 通常 < 200ms |
| 返还率 | 让球 / 胜平负 ≈ 88.5%；总进球数 ≈ 79.7%（**注意不同市场返还率差异巨大**） |
| 字段稳定性 | 已抓 4 期数据未见 schema 变更，但官方无承诺，建议定期保留 `raw/` 用于追溯 |
| 时区 | API 返回字符串本地北京时间；`captured_at` 也按本地时区写入（含 `+08:00`） |

## 十、风险提示（请认真看）

1. **赔率 ≠ 真实概率**：赔率含庄家利润（"水"），隐含概率需要打折扣后才接近真实概率。
2. **本项目只做数据 + 分析**：不构成任何"必中"承诺，不是投资建议。
3. **金钱风险自担**：投注前请严格控制预算，亏损部分必须能接受。
4. **API 使用合规**：本工具仅做个人数据归档与分析，**请遵守体彩官网 ToS 与所在地法规**，请勿用于高频商业抓取或转售数据。
