# 回测样本说明（T-3-2）

> **2026-06-16 结论**：`matches.jsonl` **不是本届世界杯的验收工具**。

## 为什么不能拿 2026 世界杯赛果回测

| 问题 | 说明 |
|---|---|
| 样本太少 | 整个赛程就几十场，统计上不显著 |
| 赛果已知 | 用已知比分验预测 = **数据泄露**，不能证明「赛前能不能猜对」 |
| 缺赛前快照 | 我们没有「开球那一刻的赔率 + 基本面 + 预测」历史档案 |

## 当前正确的验证方式（A 档）

1. **开球前**：`python3 daily_simulate.py --day <code>` → 写入 `simulation/sim_*.json`
2. **开球后**：填入实际比分，结算命中（待 T-3-3b 自动化）
3. **看过程**：方案是否有逻辑、异常场景是否体现在 `logic` 里

## `matches.jsonl` 还剩什么用

- **开发调试**：跑通 `backtest/run_backtest.py` 管道
- **日后 B 档**：2026 赛后若能从 `simulation/` + `odds_*` 时间线重建「赛前快照」，再谈 100 场档案回测

## 种子数据

30 场 2022 WC / 2024 Euro **仅作管道测试**，其中的 `features` 是简化占位，**不代表真实赛前状态**，命中率数字（如 16.7%）**无业务解读价值**。
