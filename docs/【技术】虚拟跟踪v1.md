| 字段 | 值 |
|---|---|
| 版本 | v1.0 |
| 对应 PRD | T-5-5 · T-4-11 · C 档-5 |
| 状态 | ✅ |

## 目标

赛后用统一虚拟投注口径，对比 **主推 / Top6 其余 / 被砍样本** 的收益，回答「EV 闸长期是否划算」——不做 ROI 硬门槛（样本 < 20 期）。

## 模块

| 文件 | 职责 |
|---|---|
| `virtual_tracking_lib.py` | 命中判定、虚拟盈亏、Markdown 段 |
| `virtual_tracking.py` | CLI + `tracking.md` 合并 |
| `virtual/runs/*.json` | 结算归档 |
| `tracking.md` | 「虚拟跟踪」段（`--update-tracking`） |

## 命中规则

| play_type | 规则 |
|---|---|
| `wdl` | 实际胜平负 = pick |
| `hcap` | `adj_home = home + handicap` 后比胜平负 |
| `total` | 总进球 = pick；`7+` 为 ≥7 |
| `score` | 精确比分（`胜其它` 等 v1 跳过） |
| `parlay` | 各腿均命中 |

## 样本选取

- **主推**：`funnel.hero` 1 条
- **Top6**：rank 2–6
- **被砍**：`status=rejected` 中 EV 最高的 N 条（默认 10）

## 命令

```bash
python3 virtual_tracking.py \\
  --scan validation/drafts/scan_260614.json \\
  --scores "周日010:2:2,周日012:5:1" \\
  --virtual-stake 50 \\
  --update-tracking
```

## 260614 演示结论

| 类别 | 虚拟收益 |
|---|---|
| 主推 S1（010 让球[-1]主胜） | −50 |
| Top6 其余 | −64.5 |
| 被砍样本（10 条） | **+1300**（含 012 总进球 6 球 @29 命中） |

同期真实下注 +1.40 元（B1 平命中）。**单期被砍命中 ≠ 应放开 EV 闸。**
