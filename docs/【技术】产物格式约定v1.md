| 字段 | 值 |
|---|---|
| 版本 | v1.2 |
| 实现 | `artifact_lib.py` · `report_merge_lib.py` |
| 状态 | ✅ 已落地 |

## 原则

1. **过程文档 = JSON**：Agent / 脚本之间只传递结构化 JSON，避免双写 Markdown。
2. **视图从 JSON 派生**：HTML（及可选 MD）必须在 JSON 落盘之后，由 `*_from_json()` 生成。
3. **人类交付 = 单份合并报告**：promote 时将预测 + 决策 HTML 合并为 `reports/report_*.html`。
4. **复盘用 MD**：promote 可由 JSON 生成 `reports/decision_*.md` 供复盘 Agent 读注单。
5. **统一目录**：人类报告在 `reports/`；采集快照在 `snapshots/`（回测在 `reports/backtest/`）。

## 快照目录（`snapshots/`）

| 子目录 | 内容 | 写入方 |
|---|---|---|
| `snapshots/odds/` | `odds_*.json` · `odds_window_24h_*.json` | `fetch_odds.py` |
| `snapshots/prediction/` | `prediction_*.json` | `predict_engine.py` |
| `snapshots/fundamentals/` | `fundamentals_*.json` · `fundamentals_db.json` · `teams/` | `fetch_fundamentals.py` |
| `snapshots/diffs/` | `diff_*.json` | `fetch_odds.py` |
| `snapshots/raw/` | API 原始响应 | `fetch_odds.py --save-raw` |

`prediction.meta.odds_source` 使用相对路径（如 `snapshots/odds/odds_….json`）。`latest_*` 优先查 `snapshots/`，兼容根目录遗留文件。

## 产物分层

| 层级 | 格式 | 示例 | 说明 |
|---|---|---|---|
| **源数据** | JSON | `snapshots/odds/*.json` · `snapshots/fundamentals/*.json` | 采集快照，唯一事实源 |
| **过程** | JSON | `snapshots/prediction/*.json` · `scan_*.json` · `validation/drafts/decision_*.json` | 流水线中间态 |
| **草案视图** | HTML | `validation/drafts/decision_*.html` | 门禁用，从 JSON 派生 |
| **人类交付** | HTML | `reports/report_*.html` | promote 合并预测+决策 |
| **复盘交付** | MD | `reports/decision_*.md` | 复盘 Agent 读注单 |
| **回测报告** | JSON | `reports/backtest/backtest_*.json` | 引擎管道调试 |
| **台账** | MD | `tracking.md`（**本地私有，不入 Git**）· `tracking.md.example`（模板）· `improvements_backlog.md` | 人类审批 |

## 生成链路

```
阶段2  snapshots/prediction/prediction_*.json（默认不写独立 HTML）

阶段3  decision_*.json（validation/drafts）
         └─► decision_*.html（草案视图，门禁锚定）

promote（最后一步）
  prediction_*.json ──┐
                    ├─► reports/report_*.html  ← write_merged_report()
  decision_*.json  ──┘
  decision_*.json  ──► reports/decision_*.md（复盘）
```

## CLI 开关

| 开关 | 默认 | 作用 |
|---|---|---|
| `--emit-md` | 关 | 从 JSON 额外派生 Markdown |
| `--emit-html` | 关 | 阶段2 额外写独立 `prediction_*.html`（调试用） |
| `--promote` | — | 生成合并 `report_*.html` |

## 合并报告结构

`report_<day>_workflow_<ts>.html` 含：

1. **一、赛事预测** — 五维评分 + 组合方案（来自 `prediction_*.json`）
2. **二、投注决策** — PM 工作流 + 主推 + Top3 + 行动卡（来自 `decision_*.json`）

实现：`report_merge_lib.merge_report_html()` · `write_merged_report()`
