## 目标

正式 `prediction_*` / `decision_*` 发布前，用可复算规则阻断 Agent 幻觉与数据偏离；验证产物与正式报告目录隔离。

## 模块

| 文件 | 职责 |
|---|---|
| `validation/README.md` | 目录约定、命令、禁止事项 |
| `validation_lib.py` | manifest、checks 生成；草案校验读 `draft_json` |
| `validate_gate.py` | CLI：校验 + `--promote` / `--promote-prediction` |

## 已实现必检项

- `inputs_present` — 输入文件存在
- `reference_chain` — `prediction.meta.odds_source` 与 `--odds` 一致
- `entity_anchor` — prediction 场次 ⊆ odds
- `team_names_anchor` — 主客队中文名一致
- `strategy_config` — STRATEGY_DEFAULT.yaml 含 EV/预算字段
- `draft_match_mention` — 草案场次号 ⊆ odds（有草案时）
- `draft_odds_mention` — 草案 `@x.xx` 赔率 ⊆ odds（有草案时）

## 待扩展（T-3-10 余量）

## promote（T-3-11）✅

`--promote`：PASS 后将 `prediction_*.json` + `draft-json` 合并为 **`reports/report_*.html`**，并生成 `reports/decision_*.md`。

`--promote-prediction`：在 prediction JSON 的 `meta` 写入 `validation_run_id`、`validation_promoted_at`、`validation_run_dir`。

```bash
python3 validate_gate.py \\
  --odds odds_260616_20260616-212759.json \\
  --prediction prediction_260616_20260616-214115.json \\
  --draft-json validation/drafts/decision_draft.json \\
  --promote --promote-prediction --force
```

合并报告：`report_merge_lib.write_merged_report()` · 产物格式：`docs/【技术】产物格式约定v1.md`
